Подготовительные мероприятия

Сначала изменим дату на нашем стенде. Для этого заходим на стенд и слева нажимаем Manage. Во вкладке System переходим в Time & date и жмём Edit settings.

Далее нажимаем на календарь и меняем дату на 01.01.2026. Не забываем сохранить изменения.

Переходим в Networking и создаём 4 портовые группы: gl, fil, meg, int.
Обязательно указываем VLAN ID. Он должен быть разный у каждой группы.

Переходим в Virtual Machines. Нажимаем правой кнопкой мыши (ПКМ) по каждой машине. Нажимаем Edit settings. Меняем Network Adapter в соответствии со схемой.
Для машин hw-1, hw-2, hw-3 удаляем третий и четвёртый адаптер. У данных ВМ первый интерфейс смотрит в локальную сеть. Второй интерфейс - это int.

Если мы зайдём в Networking, а затем в vSwitch0, то увидим такую картину.

Запускаем все машины кроме hw-2 и hw-3. При запуске нажимаем I Copied It. 

Для удобства дальше перейдём в программу VMware Workstation Pro (если такая есть на компьютере).
Далее нам необходимо на каждую машину (кроме Net1-AdminCA и Net3-Admin) перенести нужные файлы и программы.
Для этого сначала создадим диск, куда мы будем скидывать все папки и файлы.
Кликаем ПКМ по Net1-AdminCA, переходим в Параметры. Нажимаем Добавить, выбираем Жесткий диск.

Далее кликаем ПКМ по Моему компьютеру, переходим в Управление. Заходим в Управление дисками. Инициализируем диск. 

Кликаем ПКМ по Диску 1 и Создаём простой том. Везде нажимаем Далее ничего не меняя.

Теперь в Проводнике появился новый диск, куда мы должны скинуть все необходимые папки.
Для выполнения заданий нам понадобятся папки: Лицензия, ViPNet Administrator, Soft_ViPNet, Publication Service, Registration Point.

После завершения копирования необходимо обязательно отключить диск. Сделать это можно нажав ПКМ по Net1-AdminCA, заходим в Параметры. Нажимаем по второму диску и удаляем его. Обязательно нажимаем ОК.

После нам нужно перенести программы и файлы на другие машины.
В будущем нам придётся много раз переносить файлы с одной ВМ на другую. Поэтому запоминаем, как это делать.
Продемонстрирую это на Net1-Open. Нажимаем ПКМ по машине, заходим в Параметры. Добавляем Жесткий диск. На этом шаге необходимо использовать существующий виртуальный диск. 

Далее нажимаем Обзор и выбираем Найти в: localhost.localdomain. Два раза тыкаем по [datastore1]. После два раза жмём по Net1-AdminCA. Выбираем shared-folder.vmdk, нажимаем Открыть, а после Готово. Обязательно сохраняем изменения, нажав кнопку ОК.

Заходим в Проводник, там появился наш Жесткий диск. Переносим на Рабочий стол необходимые папки. На Net1-Open нам будут нужны: Лицензия, ViPNet Administrator.
Нужно перекидывать всё сразу, а не по одной папке, ибо есть риск повреждения файлов.
После окончания загрузки обязательно опять переходим в Параметры этой машины и удаляем диск. Нажимаем ОК.

Всё тоже самое выполняем на Net1-OperCA, Net2-Client, Net3-Open.
На Net1-OperCA нужны Soft_ViPNet, Publication Service, Registration Point.
На Net2-Client нужен Soft_ViPNet.
На Net3-Open нужны Лицензия, ViPNet Administrator.
После этого необходимо на всех машинах (пока кроме координаторов) настроить IPv4 в соответствии со схемой.  Для этого нажимаем WIN + R (или же в пуске Выполнить), пишем ncpa.cpl, открываем Свойства Ethernet0, нажимаем 2 раза по IP версии 4. Заполняем IP, маску и шлюз. 

Сразу на каждой машине нужно отключить обновления, ибо потом ВМ обновятся и придётся ооочень долго ждать. Для этого нажимаем WIN + R и пишем services.msc, ищем в самом низу службу Центр обновления Windows. Кликаем 2 раза по ней, меняем Тип запуска на Отключена. Применяем изменения и только потом нажимаем Остановить. Нажимаем ОК.

Если в будущем будет появляться такое окно, нужно нажать Подождать час. Не волнуемся, через час машина не обновится, так как у нас отключена служба, которая отвечает за обновления.

Модуль 1
Задача 1.1 Установить базу данных MSSQL на виртуальную машину Net1-Open.

Действия, описанные в данном разделе, происходят на ВМ Net1-Open.
Переходим по пути, указанному на скриншоте и устанавливаем SQL. Пусть оставляем по умолчанию.

После распаковки временных файлов автоматически запустится Установочный центр. В открывшемся окне нажимаем New SQL Server...

Принимаем пользовательское соглашение, ставим галочку напротив соответствующего пункта.

Использовать обновления Microsoft для проверки обновлений не нужно.

Не обращаем внимание на описанную установщиком ошибку. 

Необходимо установить полный комплект ПО, предлагаемый установщиком, поэтому галочки убирать не нужно.

При создании нового Instance необходимо указать имя WINNCCSQL (это имя используется по умолчанию продуктом ViPNet Administrator).

На следующем этапе установки выбираем тип Automatic напротив графы SQL Server Browser.

Необходимо выбрать пункт Mixed Mode... при выборе режима аутентификации, установить пароль P@ssw0rd. Не спешим здесь жать кнопку Далее.

В разделе FILESTREAM необходимо проставить все галочки во избежание проблем на поздних этапах развёртывания сети.

Если мы всё сделали правильно, по завершении установки увидим все зелёные галочки.

Необходимо в пуске найти SQL Server 2014 Configuration Manager, развернуть список SQL Server Network Configuration и во вкладке Protocols for WINNCCSQL включить TCP/IP.

Будет получено предупреждение о том, что для применения изменений необходимо перезапустить службу.

Для этого переходим во вкладку SQL Server Services и перезапускаем SQL Server (WINNCCSQL).

Модуль 2
Задача 2.1 Развёртывание ПК Administrator в качестве центра сертификации.

Развёртывание всех компонентов в этой задаче, кроме ViPNet Administrator ЦУС (клиентская часть) производится на ВМ Net1-AdminCA.
Запускаем установщик ViPNet Administrator ЦУС (серверная часть).

Соглашаемся с установкой необходимых компонентов.

Выбираем язык программы русский.

Принимаем условия лицензионного соглашения.

Следующим шагом необходимо проверить доступность базы данных.  

После нажатия кнопки Установить сейчас должна начаться установка продукта.

Если всё сделано правильно, будет получено сообщение Сервер успешно установлен.

Для предотвращения проблем запуска серверной части ЦУС необходимо перейти в Службы через меню пуск или по имени services.msc в WIN + R.
Здесь необходимо найти службы NccService и NccFilewatcherService, перейти в их свойства. Обязательно меняем тип запуска на Автоматически.

Следующий важный шаг - установка ViPNet Administrator УКЦ. 

Соглашаемся с лицензионным соглашением.

Ничего не изменяя, нажимаем кнопку Установить сейчас.

Следующее действие производится на ВМ Net1-Open.
Устанавливаем Клиентскую часть ЦУС.

Выбираем русский язык. 

Принимаем условия лицензионного соглашения.

Не изменяя параметров установщика, нажимаем кнопку Установить сейчас 

Если всё сделано правильно, после запуска ViPNet Administrator ЦУС (клиентская часть) мы получим ошибку Не удалось установить соединение с сервером.
Ошибка возникает, потому что клиент не имеет информации о сетевом расположении серверной части приложения.
Указываем IP-адрес Net1-AdminCA.

Если всё сделано правильно, сервер запросит имя пользователя и пароль.

  • Имя пользователя: Administrator

  • Пароль: Administrator

В следующем диалоговом окне задаём новый пароль P@ssw0rd (текущий пароль всё ещё Administrator).

Выбираем первый файл с лицензией, содержащий в себе число 24581. На экзамене номер может быть другим, главное выбрать именно первую лицензию.

Если всё сделано правильно, должно открыться меню Начало работы с.... Не нажимая ничего, закрываем.

На время оставим ЦУС открытым, перейдём на ВМ Net1-AdminCA.

Запустим установленный нами ранее ViPNet Administrator УКЦ.

Не надо ждать загрузки! Инициализация происходит только во время движения курсора мыши или нажатия любых клавиш на клавиатуре.

После инициализации указываем IP-адрес базы данных, как на скриншоте ниже.

Имя и пароль пользователя оставляем стандартными.

Далее всё, как на скриншотах. Мы не меняем параметры сертификата, поскольку далее будет создаваться новый Квалифицированный сертификат.

На данном этапе инициализации УКЦ необходимо выбрать Собственный пароль. Это очень важно! 

Задаём пароль администратора P@ssw0rd.

Если всё выполнено правильно, в завершение процесса будет получено сообщение Инициализация успешно завершена.

Модуль 3
Задача 3.1 Создание структуры защищённой сети.

Переходим в ЦУС на Net1-Open, раздел Координаторы, нажимаем знак плюс для создания нового координатора.

Указываем имя сетевого узла Корневой координатор, обязательно убрав галочку Создать одноимённого пользователя...

Аналогично создаём Подчиненный координатор.

Теперь переходим в раздел Клиенты и нажимаем зелёный знак плюс.

Первым критически важно создать именно клиента Администратор ИБ. В качестве координатора выбираем Корневой координатор, убираем галочку с пункта Создать одноименного пользователя...

Аналогично необходимо создать клиентов: Удаленный клиент, Узел ЦР. Для клиентов сети Net1 выбирать Корневой координатор, для клиентов Net2 выбирать Подчиненный координатор.

По итогу должна сложиться следующая картина.

Перейдём в раздел Пользователи и начнём создавать первого пользователя 

Создаём пользователя Root_Coordinator и присваиваем ему сетевой узел Корневой координатор.

Создадим таким же образом остальных пользователей: Sub_Coordinator, AdminS, Node_CR, Rem_Client.

Результат должен выглядеть следующим образом.

Переходим в свойства пользователя AdminS.

Затем переходим в пункт Связи с пользователями и нажимаем кнопку Добавить => Связи с пользователями.

Выбираем пользователей, представленных на скриншоте, для формирования связей и нажимаем Добавить. 

Должно получиться следующее.

Повторим действие для всех пользователей, расставляя связи в соответствии с таблицей.

Возвращаемся к разделу Координаторы и переходим в свойства сетевого узла Корневой координатор.

В разделе Межсерверные каналы добавляем узел Подчиненный координатор.

Переходим к разделу Роли узла и добавляем роль Coordinator VА-1000.HW-VA 

Переходим в раздел Адреса во внешних сетях и нажимаем Добавить, затем указываем IP-адрес координатора, направленный во внутреннюю сеть ЦО.

Выбираем IP-адрес и нажимаем кнопку Использовать для MFTP.

Добавляем так же IP-адрес, смотрящий во внешнюю сеть (Интернет для координаторов).

Аналогично повторяем все описанные действия в свойствах сетевого узла Подчиненный координатор.

Адреса во внешних сетях на подчиненном координаторе.

Для всех клиентов необходимо так же указать Адреса во внешних сетях по примеру клиента Администратор ИБ.

Роли узла для Администратор ИБ.

Роли узла для Удаленный клиент.

Роли узла для Узел ЦР. Исключительно для клиента Узел ЦР необходимо обязательно добавить роль Registration Point.

Теперь необходимо создать справочники.

Нажимаем кнопку Создать для всего списка.

Если всё сделано правильно, напротив сетевых узлов появится надпись Ожидаются ключи.

Задача 3.2 Провести инициализацию УКЦ, сохранить контейнер ключей администратора в общей папке, поменять тип паролей для пользователей.

Открываем Корневой координатор.

Переходим во вкладку Пароль администратора и нажимаем кнопку Создать пароль.

Указываем стандартный пароль - P@ssw0rd.
Описанное действие необходимо повторить так же для Подчиненного координатора.
Выбираем все сетевые узлы из списка и нажимаем Выдать новый дистрибутив ключей.

Выставляем настройки, как на скриншотах.

Пароль для всех сетевых узлов - P@ssw0rd.

Повторяем описанные действия для каждого сетевого узла!
Если всё сделано правильно, после выдачи дистрибутива ключей конечному сетевому узлу откроется проводник Windows в директории C:\ProgramData\InfoTeCS\ViPNet Administrator\KC\Export\Dst for network 24581. Опускаемся на директорию ниже.

Копируем папку Dst for network 24581 на Рабочий стол.
Статус ключей сетевых узлов в УКЦ должен смениться на Переданы в дистрибутиве ключей.

Теперь нужно скинуть папку с ключами на ранее созданный нами Жёсткий диск. Далее необходимо перенести ключи на соответствующие машины.
После этого на Net1-AdminCA открываем свойства IPv4 и меняем данные.
IP-адрес: 169.254.241.5
Маска подсети: 255.255.255.0
Основной шлюз: 169.254.241.1

В Параметрах машины меняем интерфейс на int.

Запускаем Net1-Coord, авторизуемся с учётными данными user:user и выбираем режим установки 2) full-screen interface.

Принимаем лицензионное соглашение.

Соглашаемся с инициализацией координатора.

Выбираем Europe.

Далее выбираем Russia.

После выбираем Moscow area.

Проверяем информацию и соглашаемся.

Ставим дату и точное время (секунды не нужно) как на машине Net1-AdminCA.

Далее с помощью пробела выбираем tftp.

На машине Net1-AdminCA заходим в программу Tftpd64. Переходим во вкладку Tftp Client.
Server Interfaces: 169.254.241.5
Host: 169.254.241.1
В Local File выбираем ключ Корневого координатора.

Нажимаем Put. Должно обязательно появиться такое окно.

Обратно возвращаемся на Net1-Coord и нажимаем Enter. Там появился наш ключ, нажимаем Enter.

Вводим пароль P@ssw0rd.

Настраиваем eth0. Выбираем UP. Далее StaticIP. Ставим внутренний айпи.

Теперь нужно указать шлюз. Вписываем внешний айпи второго координатора.

DNS выключаем, NTP выключаем. В названии указываем Net1-CoordCA. IP address range выключаем. Далее отказываемся от проверки соединения с другим хостом. После соглашаемся с запуском VPN-сервисов. Затем завершаем настройку.

Запускаем Net2-Coord и все описанные шаги необходимо повторить и для этого координатора!
Не забываем изменить на Net1-AdminCA Local File на Подчиненный координатор.
Сетевые параметры Net2-Coord.

Теперь ставим на Net1-AdminCA правильный айпи и интерфейс gl.

На Net1-AdminCA, Net1-OperCA и Net2-Client запускаем приложение ViPNet Client. Выходит предупреждение, о котором мы знаем, жмём ОК.

Экспортируем ключ на Рабочий стол.
